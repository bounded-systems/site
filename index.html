Bounded Systems

Give your AI agent one door, not all your keys.

Your coding agent already writes code, runs commands, and pushes to git — and today it does all of that with your access. Bounded Systems puts one sanctioned door in front of each kind of system power. The agent knocks; a broker behind the door checks your policy, acts or refuses, and records what happened.

agent → execGit({ subcommand: "push", role: "keeper" })
  ✓ pushed · keeper holds git-write · signed · audited
agent → execGit({ subcommand: "push", role: "scout" })
  ✗ refused · scout holds no git-write · recorded

01 / How it works

One door per kind of power

Each kind of system power — the filesystem, subprocesses, git, your tools — sits behind a single door. The agent holds a socket to the door, never the keys behind it. A broker daemon checks your policy on every knock, then acts or refuses and writes it down.

roomThe bundle of doors one agent holds for one job. Ask for a door it wasn't given and it fails closed — a typo never widens authority.
doorOne unit of authority: a socket to a broker that holds the real keys. The agent can knock; it cannot reach past the door.
attenuationYou narrow a door by adding caveats. Authority only ever gets tighter as it is handed inward, never looser.
guestThe agent the room hosts — for example claude-box, a Claude Code runtime. The engine itself names no particular agent.
# a room expands to exactly the doors this job holds — nothing else exists
const doors = expandRoom(rooms, catalog, "dev", env);
# narrowing only ever tightens — caveats are append-only
const narrowed = attenuate(doors[0], ["host=github.com"]);

Today prx wires these doors in-process, as library imports. The work in progress moves them onto guest-room, where each door is a real socket into a separate broker — the same model, now enforced across a process boundary.

Other agent harnesses make the work cheaper or the output leaner. This one makes it accountable — the authority layer, designed to run alongside the rest.

02 / Get started

Try it

Bounded Systems is built in the open and source-available. Pick the runtime that fits how you already work, then read the repo for the full setup.

claude-box Start here if you use Claude Code

Run Claude Code inside a room, so its authority is only the doors you hand it.

# macOS — one command sets it up
nix run .#setup
# then, in another terminal, open Claude Code into the dev room
DOORS_TCP=1 claude-box --room dev --repo .
Full setup on GitHub ↗

03 / The code

The proof is the code

Three repositories carry the model from idea to running software.

prx Flagship
The agent-run work-unit CLI
Capability-scoped agents drive each work unit to a merged PR. Git-writes are signed and verified against their owner today; the other doors are wired in-process.
Read prx ↗
guest-room The model
The door runtime prx is moving onto
A guest-agnostic capability runtime: rooms, doors, append-only attenuation, lease-bound confinement. Its specs run against the engine, so the docs can't drift from the code.
Read guest-room ↗
claude-box The guest
Claude Code, plugged into a room
The first guest: a Claude Code runtime whose authority is the doors it holds — keeper, scout, concierge, net — and nothing more. Everything else is denied and recorded.

04 / Kept honest

Every claim here is graded against the running code.

This is a working project, not a finished one. Rather than gloss over the gaps, each claim carries a grade so you can see what's proven, what's partial, and what's still a bet.

Enforced

A test, a signature, or a CI gate proves it on every commit. If it breaks, the build breaks.

Partial

True on the common path, with a named gap. The gap is written down, not glossed over.

Aspirational

The bet, not yet the result. Stated as a direction so it can't pass for done.

Enforced Docs generate from source and fail CI on drift.
Enforced guest-room's behaviour specs run against the engine.
Partial A git-write carries signed, content-addressed provenance, checked at the merge gate. — the sign-then-verify loop is tested but opt-in today; reads and egress are policy-gated, not yet signed.
Partial The agent never holds the credential — a broker daemon does. — claude-box runs credential-free; isolation today is the container plus in-process discipline, not yet a hardened sandbox.
Aspirational The guest-room door runtime replaces prx's in-process seams. — guest-room v0 ships the model; the cutover, and signing across every door, is the work.
Aspirational Agent-authored components stay honest against each other as they evolve. — the open problem this whole project is aimed at.
